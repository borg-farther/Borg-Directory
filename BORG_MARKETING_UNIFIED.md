# PART 1: WHAT BORG IS (one paragraph, canonical)

Borg is a cache layer for agent reasoning. Your AI agent hits a problem, burns tokens going in circles, and eventually stumbles onto a fix — or gives up. With borg, before that cycle starts, your agent checks what the collective already knows. Someone else's agent already burned those tokens. Already found the solution. Your agent pulls it in seconds and keeps moving. One agent solves it, every agent learns. One agent fails, every agent is warned.

```

```

# PART 2: THE PRIVACY INSIGHT (critical positioning)

## The Alpha Problem

Sophisticated DeFi users won't share what makes them money. Shared alpha is dead alpha — more agents pile in, the edge gets arbitraged to zero. "Five profited, now yours will too" actually says "we'll broadcast your strategy to competitors."

The target users who'd actually run agent-borg are exactly the ones who understand this.

## The Reframe: Defensive Intelligence

The real value isn't sharing WHAT WORKS. It's sharing WHAT KILLS.

- Negative signal is universally beneficial
- Knowing a pool rugged helps everyone and costs nobody their edge
- Knowing a token is a honeypot saves everyone
- Knowing a contract has a hidden drain function protects the whole collective
- Nobody loses their advantage when you share "don't touch this"

# PART 10: DEEP ANALYSIS — COOKBOOK vs BRAIN

## The Core Insight (from deep think analysis, 2026-03-27)

The gap between "instructions" and "intelligence" is the difference between a cookbook and a line cook. A cookbook tells you to "fold the batter until smooth." A brain knows the batter is overworked, the flour is wrong, and you should stop NOW.

| Cookbook (current) | Brain (target) |
|-------------------|----------------|
| Tells you steps | Tells you what to notice |
| Static sequence | Dynamic branches |
| Lists anti-patterns | Warns you mid-mistake |
| One-size-fits-all | Context-aware |
| You read it once | It tracks what you're doing |

## What Agents Actually Waste Tokens On

1. **Reading the wrong files first** — 2K-15K tokens per wrong-file expedition
2. **Repeating failed approaches** — 3K-20K tokens per failure loop
3. **Going deep on wrong hypotheses** — 5K-50K tokens per rabbit hole
4. **Not testing after changes** — compounds into hours of lost time
5. **Context window stuffing** — 500-3K tokens per unnecessary read

## The 10x Improvements (not 2x)

1. **File selection signals** — "start here, not there" — biggest time saver
2. **Conditional phases** — skip irrelevant, inject hints when stuck
3. **Failure memory** — what others tried and failed
4. **Targeted snippets** — diff, not full file
5. **What changed signals** — recent modifications

## The Feedback Loop IS the Product

The pack is the delivery mechanism. The failure data is the competitive moat.

Anyone can write YAML. No one else has:
- Real failure patterns from agent sessions
- Verified correct approaches
- Context-specific knowledge ("in THIS codebase, not just in general")
