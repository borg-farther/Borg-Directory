The EventBus in repo/events.py has a bug in unsubscribe(). When a Widget calls destroy() to unsubscribe its on_update method, it fails because bound methods create new objects each time they're accessed (self.on_update in subscribe creates a different object than self.on_update in unsubscribe). Fix unsubscribe to correctly identify and remove bound method handlers. Also make sure calling destroy() twice doesn't crash.
