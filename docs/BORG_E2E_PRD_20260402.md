# EXECUTIVE SUMMARY

Borg is a collective learning system for AI agents with a proven performance ceiling: baseline 2.2/10 вҶ’ borg-assisted 8.2/10 (272% improvement). However, the current implementation is a prototype with critical path failures. This PRD defines the work required to ship Borg as a production E2E system where agents genuinely share learned capabilities.

**Core Thesis:** Trace capture is the bottleneck. Navigation is the bottleneck within that. Fix trace capture and wiring, and the collective learning loop closes. Everything else is secondary.

## 4.2 SWE-Bench Benchmark Protocol

### 4.2.1 Evaluation Methodology

```
BENCHMARK DESIGN: SWE-bench Pre/Post Analysis
=============================================

Population: SWE-bench lite (300 tasks, representative difficulty)
Sample: 100 tasks for statistical significance
Control: Agent without Borg traces
Treatment: Agent with Borg traces (v2.1 format)

HYPOTHESIS:
H0: Borg traces provide no improvement (delta = 0)
H1: Borg traces improve pass rate (delta > 0)

SIGNIFICANCE LEVEL: alpha = 0.05
POWER: 0.80
MINIMUM DETECTABLE EFFECT: 10pp

STATISTICAL TEST: McNemar's test (paired binary outcomes)
```

### 4.2.2 Expected Results (HYPOTHETICAL вҖ” NOT MEASURED)

### 4.2.3 Acceptance Criteria

```
ACCEPTANCE: SWE-bench Benchmark
===============================

в–Ў Control group (no Borg): 20-25% pass rate (baseline)
в–Ў Treatment group (with Borg): 40-50% pass rate
в–Ў Delta: +20 to +30 percentage points
в–Ў p-value < 0.05 on 100-task sample
в–Ў Effect size (Cohen's h): вүҘ 0.5 (medium effect)

REGRESSION THRESHOLDS:
- If delta < 15pp: FAIL - ship blocked

- If delta > 20pp: PASS - ship cleared
```

## 5.2 Experiment Space

| Parameter | Options | Best Current | Hypothesis |
|-----------|---------|--------------|------------|
| **Trace format** | raw, structured, reasoning-only, minimal | reasoning-only | Less is more; signal/noise |
| **Context injection** | prepend, inject at tool, summarize | inject at tool | Timing matters |
| **Match algorithm** | BM25, semantic, hybrid, learned | semantic | Navigation > reasoning |
| **Pack aggregation** | concat, merge, select-best | select-best | Quality > quantity |
| **Share threshold** | always, confidence>0.7, confidence>0.9 | confidence>0.8 | Filter noise |
| **Learning extraction** | manual, LLM-extract, hybrid | LLM-extract | Automation scales |

## 5.3 Experiment Protocol

```
EXPERIMENT RUN: Trace Format Optimization
=========================================

1. GENERATE
   - Create 5 trace format variants
   - Sample 50 SWE-bench tasks (stratified)
   - Prepare 10-agent pool (varied skill levels)

2. RUN
   For each variant V in [raw, structured, reasoning, minimal, hybrid]:
     For each task T in sample:
       Run agent with V traces
       Record: pass/fail, latency, tool calls
   
3. EVALUATE
   Calculate: pass rate, avg latency, tool efficiency
   Statistical test: chi-square across variants

4. DECIDE
   Select variant with highest pass rate delta
   If delta < 5pp: try next parameter
   If delta > 10pp: freeze parameter

5. ITERATE
   Move to next experiment: Match Algorithm

SCHEDULE:
- 5 formats Г— 50 tasks Г— 3 runs = 750 experiments
- Estimated: 4 hours compute time
- Autoresearch runs nightly during Week 2
```

## 5.4 Optimization Targets

```
PARAMETER OPTIMIZATION ORDER
=============================

Priority 1: Trace format (highest impact, clearest signal)
  - V2 results: reasoning traces +34pp directional (n=3 pilot, NOT statistically significant вҖ” see PLUS34PP_AUDIT.md)
  - Hypothesis: minimal reasoning + navigation hints > full trace
  
Priority 2: Match algorithm (addresses bottleneck)
  - Experiment: Navigation > Reasoning
  - Hypothesis: semantic match + BM25 rerank > pure semantic

Priority 3: Injection timing (subtle but measurable)
  - Prepend: +12pp
  - Tool-level: +18pp  
  - Hybrid: +15pp

Priority 4: Pack aggregation (lowest priority)
  - Select-best: 50% of packs used effectively
  - Concat: 35% (more noise)
  - Merge: 40% (quality loss)

CONVERGENCE CRITERIA:
- Delta improvement < 2pp over 3 consecutive iterations
- Or: p-value > 0.1 (no significant improvement)
- Or: 20 iterations completed
```

### 9.2.3 Statistical Significance Test

```python
#!/usr/bin/env python3
"""
statistical_significance.py

McNemar's test for paired binary outcomes (SWE-bench pass/fail)
"""

import json
import math
from scipy import stats

def mcnemar_test(baseline_results, treatment_results):
    """
    baseline_results: list of bool (True = pass)
    treatment_results: list of bool (True = pass)
    
    Returns: dict with test statistics
    """
    assert len(baseline_results) == len(treatment_results)
    
    # Build contingency table
    # |                    | Treatment Pass | Treatment Fail |
    # | Baseline Pass      |       b        |       c        |
    # | Baseline Fail      |       d        |       a        |
    
    b = sum(b and t for b, t in zip(baseline_results, treatment_results))  # both pass
    c = sum(b and not t for b, t in zip(baseline_results, treatment_results))  # baseline pass, treatment fail
    d = sum(not b and t for b, t in zip(baseline_results, treatment_results))  # baseline fail, treatment pass
    a = sum(not b and not t for b, t in zip(baseline_results, treatment_results))  # both fail
    
    n = len(baseline_results)
    
    # McNemar's chi-square with continuity correction
    chi2 = (abs(d - c) - 1) ** 2 / (c + d) if (c + d) > 0 else 0
    
    # p-value
    p_value = 1 - stats.chi2.cdf(chi2, df=1) if chi2 > 0 else 1.0
    
    # Effect size (Cohen's h)
    p1 = (d + 0.5) / (n + 1)  # treatment rate
    p2 = (c + 0.5) / (n + 1)  # baseline rate
    cohens_h = 2 * (math.asin(math.sqrt(p1)) - math.asin(math.sqrt(p2)))
    
    return {
        'n': n,
        'baseline_pass_rate': (b + c) / n,
        'treatment_pass_rate': (b + d) / n,
        'delta_pp': ((b + d) - (b + c)) / n * 100,
        'table': {'a': a, 'b': b, 'c': c, 'd': d},
        'chi2': chi2,
        'p_value': p_value,
        'cohens_h': cohens_h,
        'significant': p_value < 0.05,
        'effect_size': 'small' if abs(cohens_h) < 0.5 else 'medium' if abs(cohens_h) < 0.8 else 'large'
    }

def main():
    import sys
    
    if len(sys.argv) != 3:
        print("Usage: statistical_significance.py baseline.json treatment.json")
        sys.exit(1)
    
    with open(sys.argv[1]) as f:
        baseline = [r['status'] == 'pass' for r in json.load(f)]
    with open(sys.argv[2]) as f:
        treatment = [r['status'] == 'pass' for r in json.load(f)]
    
    result = mcnemar_test(baseline, treatment)
    
    print(f"""
=== SWE-bench Statistical Analysis ===
N: {result['n']}
Baseline pass rate: {result['baseline_pass_rate']:.1%}
Treatment pass rate: {result['treatment_pass_rate']:.1%}
Delta: {result['delta_pp']:+.1f}pp

Contingency Table:
                 Treatment
                 Pass    Fail
Baseline Pass     {result['table']['b']:4d}   {result['table']['c']:4d}
Baseline Fail     {result['table']['d']:4d}   {result['table']['a']:4d}

McNemar's chi2: {result['chi2']:.3f}
p-value: {result['p_value']:.4f}
Effect size (Cohen's h): {result['cohens_h']:.3f} ({result['effect_size']})

SIGNIFICANT: {'YES' if result['significant'] else 'NO'}
""")
    
    if not result['significant']:
        print("WARNING: Result is not statistically significant (p >= 0.05)")
        sys.exit(1)
    
    if result['delta_pp'] < 15:
        print("WARNING: Delta < 15pp - may not meet acceptance criteria")
        sys.exit(1)

if __name__ == '__main__':
    main()
```
